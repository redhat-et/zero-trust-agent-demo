Zero Trust Principles Demonstrated

1. Cryptographic Workload Identity

Each service has a SPIFFE ID (e.g., spiffe://demo.example.com/user/alice) backed by X.509 certificates, not shared secrets.

2. Mutual TLS (mTLS) Everywhere

All service-to-service communication uses mutual authentication. Both client and server verify each other's identity.

3. Policy-Based Access Control

OPA evaluates Rego policies on every request. Access decisions are based on caller identity, resource requirements, and delegation context.

4. Permission Intersection

When a user delegates to an agent, effective permissions = User permissions ∩ Agent capabilities. Agents act as capability limiters.

5. Agents Cannot Act Autonomously

AI agents MUST have a delegation context from a user. Without user authorization, agents are denied access to all protected resources.

6. Short-Lived Credentials

SVIDs have 1-hour TTLs and are automatically rotated. This limits the impact of credential compromise.
